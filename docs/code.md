## 1. 괄호 (Brace) 스타일

> **K&R (Kernighan & Ritchie) 스타일**을 따릅니다.
> 
- **중괄호 올려쓰기:** 여는 중괄호 `{` 는 제어문이나 함수 선언과 **같은 줄**에 작성합니다.
- **중괄호 생략 금지:** `if`, `for`, `while` 등의 제어문 내 실행 코드가 한 줄이더라도 **무조건 중괄호를 사용**합니다.JavaScript
    
    ```
    // ✅ Good
    if (isReady) {
        startProcess();
    }
    
    // ❌ Bad
    if (isReady)
        startProcess();
    ```
    

## 2. 주석 (Comments)

> 코드는 그 자체로 의미를 전달해야 하며, 주석은 이를 보조하는 역할만 수행합니다.
> 
- **함수 헤더 주석:** 새로운 함수를 생성할 때는 해당 함수의 역할, 파라미터, 반환값 등을 설명하는 헤더 주석을 반드시 작성합니다.
- **인라인 주석 (최소화):** 코드 줄 끝에 다는 인라인 주석은 가능하지만, 꼭 필요한 경우에만 최소한으로 사용합니다.
- **불필요한 주석 금지:** 코드로 이해할 수 있는 뻔한 내용의 주석은 작성하지 않습니다.
- **TODO 태그 활용:** 추가 작업이 필요하거나 수정해야 할 부분은 `TODO` 태그를 달아 표시합니다.JavaScript
    
    ```
    // TODO: 에러 처리 로직 추가 필요
    ```
    

## 3. 함수 이름 짓기 (Function Naming)

> 함수는 특정한 **'동작'**을 수행하므로 **동사 중심**으로 시작합니다.
> 
- **데이터를 가져올 때:** `get_...`
    - *예: `get_user_id()`, `get_data()`*
- **데이터를 설정할 때:** `set_...`
    - *예: `set_status()`, `set_password()`*
- **상태를 확인할 때 (Boolean 반환):** `is_...`, `has_...`, `can_...`
    - *예: `is_empty()`, `has_token()`, `can_execute()`*

## 4. 변수 이름 짓기 (Variable Naming)

> 변수는 **'상태'**나 **'값'**을 담으므로 **명사 위주**로 짓습니다.
> 
- **의미 있는 이름 사용:** 변수명만 보고도 어떤 데이터를 담고 있는지 명확히 알 수 있어야 합니다.
- **복수형 데이터:** 데이터가 여러 개인 경우, 끝에 `s`를 붙이거나 `list`, `array` 등의 단어를 활용합니다.
    - *예: `users`, `score_list`, `item_array`*
- **임시 변수 규칙:** `for` 문 등 반복문에서 사용하는 임시 인덱스용 변수는 `i`, `j`, `k`와 같이 짧은 이름을 허용하지만, 그 외의 경우에는 반드시 의미 있는 이름을 부여합니다.
