# React Mock UI Boundary

## 지금 React mock UI 단계에서 구현한 것

- mock role 기반 STUDENT / COACH 화면 분기
- 역할별 사이드바 메뉴 분리
- 접근 불가 화면 안내 UI
- mock data 기반 게시글 목록 / 상세 연결
- mock data 기반 게시글 카테고리 필터와 검색
- mock data 기반 내 기록 카테고리 / 공개 범위 필터와 검색
- mock data 기반 게시글 작성 / 수정 / 삭제 / 댓글 작성 화면 동작
- mock data 기반 알림 드롭다운
- mock data 기반 코치 리뷰 요청 생성
- mock data 기반 대기 중 리뷰 요청 취소 시 목록에서 제거
- mock data 기반 코치 리뷰 인박스 검색 / 필터 / 상태 변경
- 코치 리뷰 상태에 `최종 확인` 추가
- GitHub repo URL 입력 후 `GitHub 프로젝트 등록` mock 동작
- 포트폴리오 프로젝트 검색
- 포트폴리오 프로젝트 카드에 포트폴리오 상태 / 코치 피드백 상태 표시
- 포트폴리오 상태를 학생이 직접 변경하는 mock UI
- 포트폴리오 프로젝트와 연결된 학습 기록 표시
- `기록 연결하기`로 내 기록 mock posts를 체크해 프로젝트에 연결
- 저장된 포트폴리오 글 초안을 핵심 정보로 표시
- README는 생성 대상이 아니라 AI 참고 자료로만 접어 표시
- AI 도우미는 포트폴리오 관리에 등록된 `내 프로젝트 선택` 기반으로만 동작
- AI 생성 결과 유형은 `포트폴리오 글`, `면접 예상 질문` 중심으로 정리
- AI 생성 결과를 `포트폴리오 초안으로 mock 저장`하는 화면 동작

## 화면 흐름 기준

```plain text
GitHub repo URL 입력
→ GitHub 프로젝트 등록
→ JungleLog 게시글 / 회고 / 트러블슈팅 / 면접 질문과 연결
→ AI 도우미에서 내 프로젝트 선택
→ GitHub 정보와 연결된 기록을 참고 자료로 확인
→ 포트폴리오 글 또는 면접 예상 질문 생성
→ 생성 결과를 포트폴리오 초안으로 mock 저장
→ 필요하면 코치 리뷰 요청
```

현재 README는 최종 산출물이 아니라 GitHub에서 가져온 참고 자료로 본다. 최종 산출물은 `포트폴리오 글 초안`과 `면접 예상 질문`이다.

## 현재 주의점

지금 화면에서 생성, 연결, 저장, 취소, 삭제, 상태 변경처럼 보이는 기능은 모두 `useState`와 mock data 기반이다.
새로고침하면 화면 상태가 초기화되며 DB에는 저장되지 않는다.
